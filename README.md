# BLC Userbot v3

## Файлы
| Файл | Назначение |
|------|------------|
| `main.py` | Ядро — приём сообщений, вся логика |
| `persona.py` | Персона Малики, шаблоны, детектор юриста/срочности |
| `ai.py` | Groq API с памятью истории диалога |
| `storage.py` | SQLite — клиенты, юзеры, история, лимиты |
| `mailer.py` | Письма юристу при горячих заявках |
| `logger.py` | Логи одновременно в консоль и Telegram-чат |
| `watchdog.py` | Автопереподключение при обрыве сессии |
| `analytics.py` | Еженедельный отчёт по понедельникам |
| `generate_session.py` | Одноразовый скрипт для SESSION_STRING |

## Логика при входящем сообщении
1. Debounce — ждём 6 сек, склеиваем сообщения от одного клиента
2. Имитация "печатает" + 4-9 сек задержка перед ответом
3. Проверка: клиент на паузе? → молчим
4. Детектор "хочу юриста" → даём ссылку, репортим в канал
5. Детектор срочности → уведомляем админа лично + логируем
6. Клиент с анкетой (has_anketa=true) → письмо юристу + репорт в канал
7. Проверка дневного лимита
8. Шаблон по ключевым словам → если не подошёл → Groq AI с историей
9. После 15 сообщений → уведомляем тебя лично, бот замолкает с этим клиентом

## Команды (пиши в личку аккаунту-менеджеру)
```
/status              — лимит, статистика, кол-во клиентов
/limit 150           — установить дневной лимит
/unpause 123456789   — снять клиента с паузы после твоего ручного ответа
/addclient 998901234567 Иван Иванов   — добавить клиента с анкетой
/report              — отправить еженедельный отчёт прямо сейчас
/help                — список команд
```

## Деплой на Railway

### 1. Получить API_ID / API_HASH
https://my.telegram.org → API development tools

### 2. SESSION_STRING (один раз локально)
```
pip install telethon
python generate_session.py
```

### 3. Railway Volume для БД
В Railway → Storage → Add Volume → Mount path: `/data`
Затем в Variables поставить `DB_PATH=/data/blc.db`
Это гарантирует что база не сбрасывается при редеплое.

### 4. Variables в Railway
Скопируй все из `.env.example`, заполни реальными значениями.

### 5. Push в репозиторий
Railway подхватит Procfile автоматически.
